# 求职助手 (JobSeek) — 产品需求文档

**版本**: v1.0  
**日期**: 2026-06-07  
**状态**: 已交付 MVP

---

## 1. 产品概述

### 1.1 产品定位
AI 驱动的一站式求职准备平台。面向求职者，覆盖简历优化 → 模拟面试 → 复盘提升的完整闭环。

### 1.2 目标用户
- **主要用户**: 正在找工作的应届毕业生及职场人士（1-5年经验）
- **次要用户**: 准备跳槽的中高级人才、想练习面试技巧的学生

### 1.3 核心价值
| 痛点 | 解决方案 |
|------|----------|
| 简历投递后石沉大海 | AI 四维度分析 + 逐条修改建议，提升简历通过率 |
| 缺乏面试实战经验 | 多类型 AI 模拟面试，7×24 随时练习 |
| 面试后不知道哪里出错 | 逐题评分反馈 + 个性化提升计划 |
| 请不起职业教练 | 全部 AI 驱动，零边际成本 |

## 2. 功能模块

### 2.1 简历修改

**入口**: 导航栏 "📄 简历修改"

**功能描述**:
用户上传简历文件或直接粘贴文本，AI 从四个维度进行深度分析，给出评分和逐条修改建议。

**输入方式**:
| 方式 | 支持格式 | 限制 |
|------|----------|------|
| 文件上传 | PDF, DOCX, DOC, TXT, MD | ≤10MB |
| 文本粘贴 | 纯文本 | 无限制 |

**分析维度**:

| 维度 | 评分 | 输出内容 |
|------|------|----------|
| 综合评分 | 0-100 | 整体评价摘要 |
| 结构分析 | 0-100 | 布局逻辑、信息完整性评估 |
| 内容优化 | 0-100 | 每段经历的原文→改进版→原因 |
| 措辞提升 | 0-100 | 措辞对比 + 优化理由 |
| 关键词匹配 | 0-100 | 已覆盖/缺失关键词列表 |

**交互流程**:
```
选择输入方式 → 上传/粘贴 → 点击"开始分析"
→ 等待 10-30 秒 → 查看五维度分析结果
→ 左侧分析报告 / 右侧原始简历对照
```

**附加功能**:
- 历史记录列表，可查看过往分析
- 支持中英文简历

---

### 2.2 模拟面试

**入口**: 导航栏 "🎯 模拟面试"

**功能描述**:
用户选择面试参数，AI 面试官逐题提问，用户回答后立即获得评分反馈。支持语音输入和 AI 虚拟形象。

**面试参数**:
| 参数 | 选项 |
|------|------|
| 面试类型 | 技术面试 / 行为面试 / 综合面试 |
| 语言 | 中文 / 英文 |
| 题目数量 | 3-10 题（滑块选择） |

**AI 面试官**:
- 6 种预设形象（3女3男，青年/中年/年长）
- 每种有独特的名字、职位头衔、渐变头像
- 三种动画状态：提问中、分析中、等待回答
- 提问时自动语音播报题目（Web Speech API）

**语音输入**:
- 浏览器原生语音识别（需 Chrome/Edge）
- 实时语音转文字填入文本框
- 支持中英文识别

**评分机制**:
- 每题即时 1-10 评分
- 每题反馈包含：优点、改进点

**交互流程**:
```
选择面试类型/语言/题数 → 自定义面试官形象（可选）
→ 开始面试 → AI 语音提问 → 语音/文字作答
→ 提交 → 查看上题反馈 + 下一题 → 重复
→ 全部完成 → 显示平均分 + 问答回顾
→ 跳转深度复盘
```

---

### 2.3 面试复盘

**入口**: 导航栏 "📊 面试复盘"

**功能描述**:
查看历史面试记录，生成深度复盘分析，包含逐题评价、参考回答、提升计划。

**复盘内容**:
| 模块 | 内容 |
|------|------|
| 综合评分 | 0-100 分 + 整体评价 |
| 优势与不足 | 各维度 list |
| 逐题分析 | 题目→你的回答→评分→评价→参考回答 |
| 短期提升 | 针对下次面试的 actionable tips |
| 长期发展 | 能力建设建议 |
| 关键要点 | 3-5 条最重要的 takeaways |

**交互流程**:
```
面试列表（按完成/进行中筛选）
→ 选择一条已完成记录
→ 点击"生成复盘分析"（首次）/ 直接查看（已生成）
→ 浏览逐题分析 + 提升计划
```

---

### 2.4 系统设置

**API Key 管理**:
- 支持 Anthropic Claude 和 DeepSeek 两种 API
- 系统根据 Key 格式自动识别（`sk-ant-` → Claude，其他 → DeepSeek）
- 可自定义 API 地址（适配代理/中转服务）
- Key 本地加密存储

## 3. 技术架构

### 3.1 技术栈

| 层次 | 技术 | 说明 |
|------|------|------|
| 前端框架 | React 18 + Vite | SPA，HMR 热更新 |
| 路由 | React Router v6 | 客户端路由 |
| 样式 | 纯 CSS | CSS 变量 + 响应式 |
| 后端 | Express.js (Node.js) | RESTful API |
| 数据存储 | JSON 文件 | 轻量级本地存储 |
| AI 引擎 | DeepSeek / Claude API | 自动适配 |
| 语音识别 | Web Speech API | 浏览器原生 |
| 语音合成 | SpeechSynthesis API | 浏览器原生 |

### 3.2 项目结构
```
job-seek/
├── server.js              # Express API 服务 (端口3001)
├── db.js                  # JSON 文件数据库
├── ai.js                  # AI 调用封装 (双引擎)
├── vite.config.js         # Vite 配置 + API 代理
├── index.html             # SPA 入口
├── src/
│   ├── main.jsx           # React 入口
│   ├── App.jsx            # 根组件 + 路由 + API Key 管理
│   ├── App.css            # 全局设计系统
│   ├── api.js             # 前端 API 封装
│   └── pages/
│       ├── Home.jsx/css           # 首页仪表盘
│       ├── ResumeRevision.jsx/css  # 简历修改
│       ├── MockInterview.jsx/css   # 模拟面试
│       ├── InterviewReview.jsx/css # 面试复盘
│       ├── AIInterviewer.jsx/css   # AI 面试官组件
├── uploads/               # 简历上传目录
└── data/                  # 数据库文件目录
```

### 3.3 API 路由

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/settings/api-key` | 保存 API Key + 地址 |
| GET | `/api/settings/api-key` | 检查 Key 状态 |
| POST | `/api/resume/upload` | 上传简历文件 |
| POST | `/api/resume/analyze` | AI 分析简历 |
| GET | `/api/resume/history` | 简历历史列表 |
| GET | `/api/resume/:id` | 简历详情 |
| DELETE | `/api/resume/:id` | 删除简历 |
| POST | `/api/interview/start` | 开始面试 |
| POST | `/api/interview/:id/answer` | 提交回答 |
| GET | `/api/interview/sessions` | 面试列表 |
| GET | `/api/interview/:id` | 面试详情 |
| POST | `/api/interview/:id/review` | 生成复盘 |

## 4. 非功能需求

| 需求 | 指标 |
|------|------|
| 浏览器兼容 | Chrome 90+, Edge 90+ (语音功能需 Chrome/Edge) |
| 响应式 | 768px 断点，移动端可用 |
| 性能 | 首屏加载 < 2s, API 响应 < 30s |
| 安全性 | API Key 本地存储，不对外发送 |
| 离线能力 | 纯前端路由 + JSON 本地存储 |
