# 面试题型分类与策略知识库

## 技术面试题
### 算法题
- 先确认需求 → 讨论多种方案 → 选最优并解释原因 → 写代码 → 测试用例 → 分析复杂度
- 评分权重：沟通40% + 解题40% + 代码质量20%

### 系统设计题
- 需求澄清 → 容量估算 → API设计 → 数据库选型 → 架构图 → 深入细节 → 权衡取舍
- 评分权重：思路50% + 深度30% + 广度20%

### 项目深挖题
- 面试官最关心的5个点：你具体做了什么、最大的挑战、如何解决的、学到了什么、再来一次会怎么改进
- 准备方式：每个项目准备3个亮点 + 2个困难 + 1个反思

## 行为面试题 (Behavioral)
### 高频题型
1. 领导力：跨团队推动、带新人、主导项目
2. 冲突解决：意见分歧、资源冲突、优先级矛盾
3. 失败反思：最大的失败、如何处理错误、复盘机制
4. 团队协作：跨部门合作、远程协作、文化差异
5. 主动性：超出职责的贡献、从0到1、创新提案

## 综合面试题
- 为什么选我们公司：研究过什么 + 认同什么 + 能贡献什么
- 职业规划：短期（1-2年）+ 中期（3-5年）+ 为什么合理
- 优缺点：真实但不致命，展示自我认知和改进行动
- 薪资期望：了解市场 + 给出范围 + 展示灵活性

## 压力面试识别与应对
- 识别信号：打断、质疑、冷淡、故意沉默
- 应对策略：保持冷静、要确认问题、结构化回答、不急于辩解
